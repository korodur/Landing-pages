# Workflow: Landing Page erstellen

## Übersicht

Dieser Workflow beschreibt den vollständigen Prozess zur Erstellung einer Landing Page mit dem Marketing-Team.

## Phase 0: Briefing (Projektleiter)

**Dauer:** Zu Beginn
**Verantwortlich:** Projektleiter

1. Anforderungen vom Nutzer sammeln:
   - Was ist das Produkt/die Dienstleistung?
   - Wer ist die Zielgruppe?
   - Was ist das Conversion-Ziel? (Signup, Kauf, Demo, Download)
   - Gibt es bestehende Brand-Guidelines?
   - Gibt es Wettbewerber-Referenzen?
   - Budget und Zeitrahmen?
2. Strukturiertes Briefing erstellen (siehe `templates/briefing.md`)
3. Briefing an alle relevanten Agents verteilen

## Phase 1: Analyse (parallel)

**Verantwortlich:** Stratege + SEO-Spezialist

### Stratege
- Wettbewerbsanalyse durchführen
- USP und Positionierung definieren
- Messaging-Framework erstellen
- Psychologische Trigger identifizieren

### SEO-Spezialist
- Keyword-Recherche durchführen
- Suchintention analysieren
- SEO-Anforderungen definieren
- Schema-Markup-Typ festlegen

**Gate:** Projektleiter prüft und gibt frei

## Phase 2: Content (sequentiell)

**Verantwortlich:** Content Creator → CRO-Optimierer

### Content Creator
- Landing Page Copy schreiben (alle Sektionen)
- Meta-Texte erstellen
- SEO-Keywords integrieren
- Ad-Texte vorbereiten

### CRO-Optimierer (Review)
- Copy auf Conversion-Potenzial prüfen
- CTA-Optimierung vorschlagen
- Seitenstruktur-Empfehlungen geben
- Trust-Signal-Platzierung empfehlen

**Feedback-Loop:** Content Creator überarbeitet basierend auf CRO-Feedback

**Gate:** Projektleiter prüft und gibt frei

## Phase 3: Umsetzung (parallel mit Abstimmung)

**Verantwortlich:** Landing Page Builder + SEO-Spezialist + CRO-Optimierer

### Landing Page Builder
- HTML/CSS/JS umsetzen
- Responsive Design sicherstellen
- Performance optimieren

### SEO-Spezialist
- Schema Markup implementieren
- Meta-Tags finalisieren
- Technisches SEO prüfen

### CRO-Optimierer
- Finale Conversion-Optimierung
- Tracking-Code einbauen
- A/B-Test-Setup vorbereiten

**Gate:** Projektleiter führt Qualitätscheck durch
- Lighthouse-Scores prüfen
- Mobile-Test
- Cross-Browser-Test
- Alle Links und CTAs testen

## Phase 4: Launch (parallel)

**Verantwortlich:** Growth Manager + CRO-Optimierer

### Growth Manager
- Traffic-Strategie aktivieren
- Ad-Kampagnen starten
- E-Mail-Sequenzen ausspielen
- Social-Media-Posts veröffentlichen

### CRO-Optimierer
- Tracking verifizieren
- Erste A/B-Tests starten
- Conversion-Daten monitoren

**Gate:** Projektleiter liefert Ergebnis-Report an Nutzer

## Checkliste vor Go-Live

- [ ] Alle Texte korrekturgelesen
- [ ] Responsive auf Mobile, Tablet, Desktop
- [ ] Lighthouse Score > 90
- [ ] Schema Markup validiert
- [ ] Analytics-Tracking funktioniert
- [ ] CTAs verlinken korrekt
- [ ] DSGVO: Cookie-Consent, Impressum, Datenschutz
- [ ] Ladezeit < 3 Sekunden
- [ ] Favicon und OG-Tags gesetzt
- [ ] 404-Seite vorhanden
