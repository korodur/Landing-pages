### 1.1 Keyword-Cluster-Übersicht

Die Keyword-Strategie für NEODUR Level operiert in einer **technischen B2B-Nische**. Das bedeutet:
- Suchvolumen pro Keyword ist gering (10--500 Suchanfragen/Monat)

- Wettbewerb in der organischen Suche ist moderat bis gering
- Content-Qualität und technische Tiefe entscheiden über Rankings

Nach Analyse der Wettbewerbslandschaft (Sika, MC-Bauchemie, BASF, Flowcrete) gibt es folgende Content-Lücken, die NEODUR Level besetzen kann:

| Content-Gap | Begründung | Chance für KORODUR |
|---|---|---|
| **Spezifischer Content zu AutoStore-Bodenanforderungen** | Kein deutscher Hersteller bietet eine dedizierte Landing Page für AutoStore/Shuttle-Böden | Erste Anlaufstelle in Google für dieses Thema werden |
| **Vergleich Dünnschichtestrich vs. konventioneller Estrich** | Existierende Inhalte sind generisch und nicht auf Automatisierungskontext bezogen | Konkreten Vergleich mit Zahlen (Zeit, CO2, Kosten) bieten |
| **Technische Anforderungen für Planer als Download** | Kaum fertige Ausschreibungstexte oder Planungsleitfäden verfügbar | PDF-Downloads als Lead-Magnet anbieten |
| **Sanierungslösungen für bestehende Lagerböden** | Die meisten Inhalte behandeln nur Neubau, nicht Bestandssanierung | Neubau UND Sanierung mit konkretem Systemaufbau darstellen |
| **Nachhaltigkeit quantifiziert (CO2-Reduktion)** | Wettbewerber machen vage Nachhaltigkeitsaussagen | Konkrete Zahl (30 % weniger CO2) als Differenzierung |
| **Zielgruppenspezifische Nutzenargumentation** | Inhalte sind typischerweise "one size fits all" | Vier dedizierte Argumentationsstränge je Rolle |

## 3. On-Page SEO-Anforderungen

### 3.1 Title Tag

**Empfehlung (max. 60 Zeichen):**

```
NEODUR Level | Industrieboden für AutoStore & Automation
```

**Zeichenzahl:** 57

**Alternativ-Varianten:**
- `NEODUR Level -- Industrieboden für automatisierte Lager` (54 Zeichen)
- `Industrieboden AutoStore | NEODUR Level von KORODUR` (53 Zeichen)
- `AutoStore Boden | NEODUR Level Industrienutzschicht` (54 Zeichen)

**Logik:** Der Title Tag enthält den Produktnamen (NEODUR Level), das Hauptkeyword (Industrieboden) und den Anwendungskontext (AutoStore/Automation). Die Marke KORODUR kann im Title entfallen, da sie in der Domain (korodur.de) bereits sichtbar ist -- Google zeigt die Domain automatisch mit an.

**Primäre Empfehlung:**

```
https://korodur.de/industrieboden-automatisierte-lagersysteme/
```

**Alternative Varianten:**

| Option | URL | Bewertung |
|---|---|---|
| A (empfohlen) | `/industrieboden-automatisierte-lagersysteme/` | Hauptkeyword, klar, suchmaschinen-optimiert |
| B | `/neodur-level-autostore-boden/` | Produktfokus + Anwendung, spezifischer |
| C | `/produkte/neodur-level/` | Logisch in Seitenarchitektur, aber weniger keyword-relevant |
| D | `/loesungen/autostore-industrieboden/` | Lösungsorientiert, gute Nutzerintention |

**Empfehlung:** Variante A als Haupt-URL, da sie das Hauptkeyword ("Industrieboden automatisierte Lagersysteme") exakt abbildet. Falls korodur.de eine Produktstruktur unter `/produkte/` hat, wäre Variante C mit Redirect von A als 301-Weiterleitung denkbar.

**URL-Regeln:**
- Nur Kleinbuchstaben
- Bindestriche als Worttrenner (kein Unterstrich)
- Keine Stoppwörter (für, und, der, die)
- Kein Trailing Slash-Wechsel (konsequent mit oder ohne)
- Max. 3--5 Wörter im Pfad

### 4.6 Validierungshinweise

| Schritt | Tool | Aktion |
|---|---|---|
| 1 | [Google Rich Results Test](https://search.google.com/test/rich-results) | Alle JSON-LD-Blöcke einzeln und kombiniert testen |
| 2 | [Schema.org Validator](https://validator.schema.org/) | Syntaktische Korrektheit prüfen |

**Wichtige Hinweise:**
- Alle JSON-LD-Blöcke im `<head>` oder vor `</body>` platzieren
- Keine doppelten Schema-Typen auf derselben Seite (z.B. nicht zweimal Product)
- URLs in den Schema-Markups müssen mit den tatsächlichen Live-URLs übereinstimmen
- Logo-URL und Bild-URLs müssen existierende, erreichbare Dateien sein
- Nach jedem Update: erneut validieren

## 5. Technische SEO-Checkliste

### 5.1 Core Web Vitals

| Metrik | Zielwert | Maßnahmen |
|---|---|---|
| **LCP** (Largest Contentful Paint) | < 2,5 Sekunden | Hero-Bild optimieren (WebP, Preload), kritisches CSS inline, Server-Response < 200ms |
| **INP** (Interaction to Next Paint) | < 200 Millisekunden | JavaScript minimieren, keine render-blocking Scripts, Event-Handler optimieren |
| **CLS** (Cumulative Layout Shift) | < 0,1 | Bildabmessungen (width/height) immer angeben, Schriftarten preloaden, keine dynamischen Elemente über dem Fold |

### 5.2 Mobile-First-Checkliste

- [ ] **Viewport-Meta-Tag:** `<meta name="viewport" content="width=device-width, initial-scale=1">`
- [ ] **Touch-Targets:** Alle klickbaren Elemente mind. 48x48 px
- [ ] **Schriftgröße:** Fließtext mind. 16px auf Mobile
- [ ] **Keine horizontale Scrollbar** auf allen Breakpoints
- [ ] **CTA-Button** immer sichtbar (sticky oder wiederkehrend)
- [ ] **Tabellen responsiv:** Technische Daten als Cards oder horizontal scrollbar auf Mobile
- [ ] **Bilder responsiv:** `srcset` und `sizes` für verschiedene Auflösungen
- [ ] **Formulare:** Input-Felder mit korrekten `type`-Attributen (tel, email) für mobile Keyboards
- [ ] **Hamburger-Menü** für Navigation auf Mobile
- [ ] **Kein Hover-abhängiger Content** -- alle Informationen auch ohne Hover zugänglich

### 5.3 Ladezeit-Optimierung

| Maßnahme | Priorität | Detail |
|---|---|---|
| **Bilder: WebP/AVIF** | HOCH | Alle Bilder als WebP mit JPEG-Fallback, Hero-Bild auch als AVIF |
| **Bilder: Lazy Loading** | HOCH | `loading="lazy"` für alle Bilder unterhalb des Folds |
| **Hero-Bild: Preload** | HOCH | `<link rel="preload" as="image" href="hero.webp">` |
| **CSS: Critical inline** | HOCH | Above-the-Fold CSS inline im `<head>`, Rest async laden |
| **JS: Defer/Async** | HOCH | Alle Scripts mit `defer` oder `async`, kein render-blocking JS |
| **Fonts: Preload** | MITTEL | Gabarito per `<link rel="preload">` laden, `font-display: swap` |
| **Fonts: Subset** | MITTEL | Nur benötigte Zeichen (Latin + deutsche Sonderzeichen) laden |
| **Komprimierung: Gzip/Brotli** | HOCH | Server-seitige Komprimierung aktivieren |
| **Caching: HTTP-Header** | MITTEL | `Cache-Control` für statische Assets (1 Jahr), HTML (no-cache) |
| **CDN** | NIEDRIG | Falls Traffic-Volumen es rechtfertigt |
| **Minifizierung** | HOCH | HTML, CSS, JS minifizieren |

**Ziel:** Lighthouse Performance Score > 90, Gesamtladezeit < 3 Sekunden.

### 5.4 Canonical, Hreflang, Robots

#### Canonical Tag

```html
<link rel="canonical" href="https://korodur.de/industrieboden-automatisierte-lagersysteme/" />
```

**Wichtig:** Canonical auf jeder Seite setzen, auch wenn es die einzige Version ist. Verhindert Duplicate-Content-Probleme durch URL-Parameter (z.B. UTM-Tags, Session-IDs).

#### Hreflang (falls mehrsprachig geplant)

Aktuell nur Deutsch. Falls zukünftig englische Version geplant:

```html
<link rel="alternate" hreflang="de" href="https://korodur.de/industrieboden-automatisierte-lagersysteme/" />
<link rel="alternate" hreflang="en" href="https://korodur.de/en/industrial-flooring-automated-warehouses/" />
<link rel="alternate" hreflang="x-default" href="https://korodur.de/industrieboden-automatisierte-lagersysteme/" />
```

**Empfehlung:** Hreflang erst implementieren, wenn die englische Version tatsächlich existiert. Sonst weglassen.

#### Robots

```html
<meta name="robots" content="index, follow" />
```

Die Seite soll indexiert und alle Links gefolgt werden. Kein `noindex` oder `nofollow`.

| **Featured Snippet (Tabelle)** | JA -- für Vergleiche | HTML-Tabellen für technische Daten und Vergleiche |

### 6.2 FAQ-Snippet-Optimierung

**Umsetzung in 3 Schritten:**

**Schritt 1: FAQ-Section im HTML**

Die FAQ-Section sollte semantisch korrekt aufgebaut sein. Jede Frage-Antwort als eigener Block:

```html
<section id="faq">
  <h2>Häufige Fragen zu Industrieböden für automatisierte Lagersysteme</h2>

  <div class="faq-item">
    <h3>Welche Bodenanforderungen hat ein AutoStore-System?</h3>
    <p>AutoStore-Systeme erfordern eine planebene Bodenfläche mit engen
    Toleranzen, hoher Druckfestigkeit für die Stützenlasten des Grid-Systems
    und eine staubarme Oberfläche zum Schutz der Robotik und Sensorik.
    NEODUR Level erfüllt diese Anforderungen mit einer Druckfestigkeit von
    C40 (40 N/mm²), Verschleißwiderstand AR 0,5 und selbstverlaufender,
    planebener Oberfläche.</p>
  </div>

  <!-- Weitere FAQ-Items -->
</section>
```

**Schritt 2: FAQPage Schema Markup (siehe Abschnitt 4.3)**

Das JSON-LD muss exakt den gleichen Text wie die sichtbare FAQ-Section enthalten. Google bestraft Abweichungen zwischen sichtbarem Content und Schema Markup.

**Schritt 3: Fragen-Auswahl optimieren**

Die besten FAQ-Kandidaten für Rich Results sind:
- Fragen, die exakt so in Google gesucht werden (siehe Abschnitt 2.1)
- Antworten zwischen 40--60 Wörtern (ideal für Featured Snippets)
- Fragen mit klarer, faktischer Antwort (keine Meinungen)

### 6.3 Featured-Snippet-Strategie

**Typ 1: Paragraph-Snippet**

Für Definitionen und kurze Erklärungen. Format:

```
H2/H3: Was ist [Keyword/Frage]?
Direkt darunter: 40-60 Wörter Antwort im ersten Absatz.
```

**Beispiel:**

```html
<h3>Was ist ein Dünnschichtestrich?</h3>
<p>Ein Dünnschichtestrich ist ein Industrieboden mit einer Aufbauhöhe von
nur 4–30 mm, der als Alternative zu konventionellem Estrich (40–80 mm)
eingesetzt wird. Er ist selbstverlaufend, schnell belastbar und reduziert
den Materialverbrauch erheblich. NEODUR Level von KORODUR erreicht trotz
geringer Schichtdicke eine Druckfestigkeit von C40 (40 N/mm²).</p>
```

**Typ 2: Tabellen-Snippet**

Für Vergleiche und technische Daten. Google extrahiert bevorzugt `<table>`-Elemente:

```html
<h3>NEODUR Level vs. konventioneller Estrich</h3>
<table>
  <thead>
    <tr>
      <th>Kriterium</th>
      <th>NEODUR Level</th>
      <th>Konventioneller Estrich</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>Aufbauhöhe</td><td>4–30 mm</td><td>40–80 mm</td></tr>
    <tr><td>Begehbar nach</td><td>ca. 3 Stunden</td><td>1–3 Tage</td></tr>
    <tr><td>Voll belastbar nach</td><td>ca. 3 Tage</td><td>4–8 Wochen</td></tr>
    <tr><td>Druckfestigkeit</td><td>C40 (40 N/mm²)</td><td>C25–C30</td></tr>
    <tr><td>CO₂-Bilanz</td><td>30 % weniger</td><td>Referenzwert</td></tr>
    <tr><td>Selbstverlaufend</td><td>Ja</td><td>Nein (manuell)</td></tr>
  </tbody>
</table>
```

**Typ 3: Listen-Snippet**

Für Anforderungen, Schritte und Vorteile:

```html
<h3>Bodenanforderungen für automatisierte Lagersysteme</h3>
<ol>
  <li>Planebene Oberfläche mit Millimetertoleranzen</li>
  <li>Hohe Druckfestigkeit für Stützen- und Punktlasten (min. C40)</li>
  <li>Staubarme Oberfläche zum Schutz der Sensorik</li>
  <li>Hoher Verschleißwiderstand (min. AR 1, ideal AR 0,5)</li>
  <li>Schnelle Verfügbarkeit bei Neubau und Sanierung</li>
</ol>
```
